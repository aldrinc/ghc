# Apify Integration Guide

## Overview

The Apify scraper layer sits between the Strategy Agents (Agent 0 + Agent 0b) and the Qualifier Agent (Agent 1). It executes the search strategies and configurations generated by the strategy agents and produces raw scraped data for qualification.

## Integration Modes

### Mode A: Automated (API)

Agent 0/0b can make direct HTTP calls to the Apify REST API if an API token is available in the environment.

```bash
export APIFY_TOKEN=your_api_token_here
```

API endpoint: `https://api.apify.com/v2`

**Start a run:**
```bash
curl -X POST "https://api.apify.com/v2/acts/{actorId}/runs" \
  -H "Authorization: Bearer $APIFY_TOKEN" \
  -H "Content-Type: application/json" \
  -d @apify_config.json
```

**Poll for completion:**
```bash
curl "https://api.apify.com/v2/actor-runs/{runId}" \
  -H "Authorization: Bearer $APIFY_TOKEN"
```

**Download dataset:**
```bash
curl "https://api.apify.com/v2/datasets/{datasetId}/items?format=json" \
  -H "Authorization: Bearer $APIFY_TOKEN" \
  -o raw_scraped_data.json
```

### Mode B: Manual (Console/CLI)

1. Open [Apify Console](https://console.apify.com)
2. Find the actor listed in the config
3. Paste the `input` JSON from the config file
4. Run the actor
5. Download the dataset as JSON
6. Save to the appropriate `raw_scraped_data/` subdirectory

## Supported Actors

### Text Habitat Actors

| Actor | Actor ID | Use Case | Key Input Fields |
|-------|----------|----------|------------------|
| Reddit Scraper | `trudax/reddit-scraper` | Subreddit posts + comments | `subreddit`, `sort`, `time`, `limit`, `includeComments` |
| Web Scraper | `apify/web-scraper` | Forums, blog comment sections | `startUrls`, `pageFunction`, `maxRequestsPerCrawl` |

| Amazon Reviews | `junglee/amazon-reviews-scraper` | Product reviews | `productUrls`, `maxReviews` |
| Google SERP | `apify/google-search-scraper` | Forum URL discovery | `queries`, `maxPagesPerQuery`, `countryCode` |

### Social Video Actors

| Actor | Actor ID | Use Case | Key Input Fields |
|-------|----------|----------|------------------|

## Rate Limiting & Best Practices

1. **Stagger runs** — don't launch all actors simultaneously
2. **Respect platform limits** — keep `maxRequests` reasonable (200-500 per run)
3. **Use proxies** — Apify provides residential proxies for sensitive platforms
4. **Save raw data** — always keep the unprocessed Apify output alongside normalized data
5. **Monitor costs** — track compute unit consumption per actor run
6. **Cache results** — re-running the same search within 7 days is wasteful; cache and reuse

## Error Handling

| Error | Cause | Resolution |
|-------|-------|------------|
| Actor timeout | Too many results requested | Reduce `limit`/`maxResults`, split into multiple runs |
| 429 Rate Limited | Platform rate limiting | Add delays, use residential proxies |
| Empty dataset | Search terms too specific | Broaden search, check search syntax |
| Partial results | Actor crashed mid-run | Re-run with smaller batch, check actor logs |
| Auth required | Private content | Mark habitat as GATE_FAIL in Agent 1 |

## Post-Scrape Processing Pipeline

```
raw_scraped_data/social_video/
        |
        v
  score_virality.py
    Computes: virality_ratio, velocity, engagement_density
    Outputs: scored_video_data/ (with tier labels)
        |
        v
  Agent 1: Habitat Qualifier
    Reads: raw_scraped_data/text_habitats/ + scored_video_data/
    Fills: observation sheets
    Outputs: habitat_observations.json
        |
        v
  score_habitats.py
    Scores: all habitats (text + video)
    Outputs: habitat_scores.json
```
