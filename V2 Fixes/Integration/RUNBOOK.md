# Pipeline Runbook — Operator Execution Guide

**Purpose:** Step-by-step instructions for running the complete agentic marketing pipeline from seed input to scored copy output.

**Prerequisite:** Read `docs/plans/2026-02-21-pipeline-integration-design.md` for architectural context. This runbook tells you WHAT to do; the design doc explains WHY.

---

## Human-in-the-Loop Decision Points

There are **6 mandatory human decision points** in this pipeline. The system cannot proceed past these without human judgment.

| # | Decision | When | What You Decide | Data Available | Time Needed |
|---|----------|------|-----------------|---------------|-------------|
| **H1** | Proceed past research | After Stage 1 completes | Is the research quality sufficient? | STEP1, STEP4, STEP6 outputs + quality gate results | 15-30 min review |
| **H2** | Collect competitor assets | Before Stage 2A | Which 3+ competitor assets to analyze? | Competitor names/URLs from STEP1 | 30-60 min collection |
| **H3** | Select angle | After Stage 2B completes | Which Purple Ocean angle to build the offer around? | Agent 3's 15-30 ranked angles with observation sheets, hooks, compliance, evidence | 30-60 min evaluation |
| **H4** | Select UMP/UMS pair | During Stage 3, Step 3 | Which unique mechanism pair to use? | 3-5 scored pairs with dimension breakdowns | 15-30 min |
| **H5** | Select offer variant | During Stage 3, Step 5 | Which offer variant to proceed with? | Base + 2-3 variants with 8-dimension evaluations | 15-30 min |
| **H6** | Review/approve copy | After Stage 4 | Is the scored copy ready for deployment? | Scored headlines, advertorials, sales pages with test results | 30-60 min |

Additionally, these **operator inputs** are required at specific stages:
- **Stage 0:** Fill 5 seed fields (product_name, description, price, competitor_urls, product_customizable)
- **Stage 3 entry:** Provide 6 operator-specified fields (business_model, funnel_position, target_platforms, target_regions, existing_proof_assets, brand_voice_notes)
- **Offer-to-Copy Bridge:** Configure `brand-voice.md` (not generated by pipeline)

In the step-by-step instructions below, human decision points are marked with **[HUMAN DECISION]**.

Ensure you have:
- [ ] All four system folders present in the workspace

- [ ] Apify account configured (for VOC + Angle Engine scraping)

## Stage 0: Seed Input [HUMAN INPUT]

**Time:** 5-10 minutes
**Who:** Operator

1. Create a `product_brief.json` file with these 5 fields:

```json
{
  "stage": 0,
  "product_name": "Your Product Name",
  "description": "1-3 sentence description of what the product is and does.",
  "price": "$49 one-time",
  "competitor_urls": ["https://competitor1.com", "https://competitor2.com"],
  "product_customizable": true
}
```

2. Validate against `integration/schemas/product_brief_stage0.json`
3. Derive `BUSINESS_CONTEXT` = one-sentence summary of product name + description
4. Derive `BUSINESS_CONTEXT_JSON` = the full JSON above

**Output:** `product_brief.json` (Stage 0) + `BUSINESS_CONTEXT` + `BUSINESS_CONTEXT_JSON`

**Proceed when:** All 5 fields are filled. If price is unknown, set to "TBD".

## Stage 1: Foundational Research

**Time:** 2-4 hours (mostly agent execution time)
**System:** `Foundational Docs Prompts/clean_prompts/`

### Step 1.1: Competitor Research

- **Prompt:** `01_competitor_research_v2.md`
- **Input:** `{{BUSINESS_CONTEXT}}`, `{{BUSINESS_CONTEXT_JSON}}`
- **Agent type:** Web-access research agent
- **Output:** `STEP1_SUMMARY`, `STEP1_CONTENT`, `CATEGORY_NICHE`

### Step 1.2: Deep Research Meta-Prompt

- **Prompt:** `03_deep_research_meta_prompt_v2.md`
- **Input:** `{{BUSINESS_CONTEXT}}`, `{{BUSINESS_CONTEXT_JSON}}`, `{{STEP1_SUMMARY}}`, `{{STEP1_CONTENT}}`
- **Agent type:** Prompt engineer (no web access needed)
- **Output:** `STEP4_PROMPT` (a complete research prompt)

### Step 1.3: Deep Research Execution

- **Prompt:** Execute `STEP4_PROMPT` (the output from Step 1.2)
- **Agent type:** Web-access research agent
- **Output:** `STEP4_SUMMARY`, `STEP4_CONTENT`

### Step 1.4: Avatar Brief

- **Prompt:** `06_avatar_brief_v2.md`
- **Input:** `{{BUSINESS_CONTEXT}}`, `{{BUSINESS_CONTEXT_JSON}}`, `{{STEP1_SUMMARY}}`, `{{STEP1_CONTENT}}`, `{{STEP4_SUMMARY}}`, `{{STEP4_CONTENT}}`
- **Agent type:** Strategist (no web access needed)
- **Output:** `STEP6_SUMMARY`, `STEP6_CONTENT`

### Quality Gate [HUMAN DECISION H1: Proceed past research?]

Before proceeding, verify:
- [ ] STEP1 identifies at least 3 validated competitors
- [ ] STEP4 has VOC data in at least 5 of 9 research categories (A-I)
- [ ] STEP6 identifies at least 3 buyer segments

**If gates fail:** See Error Recovery table. You may choose to proceed with caveats or rerun.

### Enrich Product Brief

Using `integration/specs/translation-layer.md` (Section: Enriching the Product Brief), add these fields to your `product_brief.json`:

- `category_niche` — from STEP1_CONTENT header
- `market_maturity_stage` — from STEP1_CONTENT market assessment
- `primary_icps` — top 3 segments from STEP6_CONTENT
- `positioning_gaps` — from STEP1_CONTENT whitespace section
- `competitor_count_validated` — count from STEP1_CONTENT

Update `stage` to `1`. Validate against `integration/schemas/product_brief_stage1.json`.

## Stage 2B: VOC + Angle Discovery

**Time:** 4-8 hours (includes Apify scraping time)
**System:** `VOC + Angle Engine (2-21-26)/`
**Reference:** `integration/specs/translation-layer.md` for all input mappings

### Step 2B.1: Transform VOC Corpus for DUAL MODE

Before running Agent 2, transform STEP4_CONTENT into Agent 2's format:

1. Follow `integration/specs/translation-layer.md` → "VOC Corpus Transformation" section
2. Extract all tagged quotes from STEP4_CONTENT
3. Map categories (A-I → 8 core dimensions) and emotions using the mapping tables
4. Assign sequential VOC IDs (V001, V002, ...)
5. Flag all items as `EXISTING_CORPUS`
6. Do NOT fill observation sheets — Agent 2 does that

Save the transformed corpus for Agent 2 input.

### Step 2B.2: Run Agent 0 (Habitat Strategist) + Agent 0b (Video Strategist)

Run in parallel. Use `integration/specs/translation-layer.md` for input mapping:

- **Agent 0 inputs:** `PRODUCT_BRIEF` (enriched), `AVATAR_BRIEF` (STEP6_SUMMARY), `COMPETITOR_RESEARCH` (STEP1), `COMPETITOR_ANALYSIS_JSON`
- **Agent 0b inputs:** Same + `PRODUCT_CATEGORY_KEYWORDS` extracted from CATEGORY_NICHE + STEP1_CONTENT

### Step 2B.3: Apify Execution

Run scraper configs from Agent 0 + 0b outputs. If video habitats were scraped, run `score_virality.py`.

### Step 2B.4: Run Agent 1 (Habitat Qualifier)

- **Input:** Agent 0 handoff block + scraped data
- **Quality gate:** At least 8 qualified habitats
- Run `score_habitats.py` on output

### Step 2B.5: Run Agent 2 (VOC Extractor) — DUAL MODE

- **Input:** Agent 1 handoff block + transformed VOC corpus (from Step 2B.1) + `PRODUCT_BRIEF` + `AVATAR_BRIEF`
- **Critical:** Must run in DUAL MODE because Stage 1 data exists
- **Quality gate:** At least 200 VOC items across 3+ habitat types
- Run `score_voc.py` on output

### Step 2B.6: Run Agent 3 (Shadow Angle Clusterer)

- **Input:** Agent 2 handoff block + `Competitor Angle Map` (restructured from competitor_analysis.json) + `Known Saturated Angles` + `PRODUCT_BRIEF` + `AVATAR_BRIEF`
- See `integration/specs/translation-layer.md` for restructuring instructions
- **Quality gate:** At least 5 candidate angles with 5+ supporting VOC each
- Run `score_angles.py` on output

## [HUMAN DECISION H3: Select Angle]

**Time:** 30-60 minutes
**Template:** `integration/templates/angle_selection.yaml`
**Guide:** `integration/templates/angle_selection_guide.md`

1. Review Agent 3's output sections in this order:
   - Section 2 (Saturated Angle Summary)
   - Section 7 (Decision Readiness Block)
   - Section 4 (Observation Sheets)
   - Section 3 (Candidate Angle Profiles)
   - Section 5 (Overlap Matrix)

2. Apply selection criteria from the guide:
   - **Hard gates:** VOC count >= 5, compliance expressible without RED, no INSUFFICIENT_EVIDENCE flag
   - **Strong signals:** MULTI triangulation, 3+ distinctiveness, ACCELERATING/STEADY velocity
   - **Red flags:** SINGLE_SOURCE_RISK, HIGH_VARIANCE, DECELERATING

3. Fill `angle_selection.yaml` using the field-by-field source map

4. Update `product_brief.json` with Stage 2 fields (selected_angle, compliance_constraints, etc.). Set `stage` to `2`. Validate against `integration/schemas/product_brief_stage2.json`.

## Stage 3: Offer Architecture

**Time:** 2-4 hours
**System:** `Offer Agent — Final/`
**Reference:** `integration/specs/offer-agent-input-mapping.md` for all input assembly

### Prepare Inputs [HUMAN INPUT: 6 operator-specified fields]

Using `integration/specs/offer-agent-input-mapping.md`, assemble the 5 inputs:

1. **product_brief** — Your Stage 2 product_brief.json (note field name translations in the mapping spec)
2. **selected_angle** — From angle_selection.yaml, transformed per the mapping spec
3. **competitor_teardowns** — From competitor_analysis.json (include/exclude per spec) + relevant STEP1_CONTENT sections
4. **voc_research** — Agent 2 corpus, **filtered to selected angle** (40-80 items per filtering rules in spec)
5. **purple_ocean_research** — Complete Agent 3 handoff (ALL angles, not just selected)

**Operator-specified fields** (not from upstream — you must provide):
- `business_model` (one-time / subscription / freemium)
- `funnel_position` (cold_traffic / post_nurture / retargeting / evergreen)
- `target_platforms` (Meta / TikTok / YouTube / Google / Email)
- `target_regions` (US / UK / etc.)
- `constraints.existing_proof_assets`
- `constraints.brand_voice_notes`

### Run the Pipeline

Follow `Offer Agent — Final/prompts/pipeline-orchestrator.md`:

1. Step 1: Avatar Brief synthesis
2. Step 2: Market Calibration (7 phases) — produces awareness-angle-matrix
3. Step 3: UMP/UMS Generation → **[HUMAN DECISION H4: SELECT your UMP/UMS pair]**
4. Step 4: Offer Construction (13 phases, 3 scoring tools)
5. Step 5: Self-Evaluation → **[HUMAN DECISION H5: SELECT your offer variant]**

### Quality Gate

- Step 2 `calibration_consistency_checker` must pass
- Step 5 composite score >= 5.5 (PASS)
- If < 5.5, iteration logic runs (max 2 iterations before HUMAN_REVIEW)

### Update Product Brief

Add Stage 3 fields (ump, ums, core_promise, etc.). Set `stage` to `3`. Validate against `integration/schemas/product_brief_stage3.json`.

## Stage 4: Copy Execution

**Time:** 3-6 hours
**System:** `Copywriting Agent — Final/`
**Ends with:** [HUMAN DECISION H6: Review/approve copy]

### Run Copy Pipeline

1. Load all shared context files (they should now be fully populated)
2. Run headline engine → `headline_scorer_v2.py` (29 tests, 44 pts)
3. Extract Promise Contracts from winning headlines
4. Write presell advertorial → congruency scoring (13 tests, 19 pts)
5. Write sales page → congruency scoring
6. QA auto-fix loop for failing assets (max 3 iterations)
7. Generate output documents

### Quality Gates

- [ ] Headlines score B-tier or above (28+ points)
- [ ] Presell congruency passes 75% (14.25+ of 19 points)
- [ ] Sales page congruency passes 75%
- [ ] PC2 (Promise Contract Delivery Test) passes — this is a HARD GATE

### If Quality Gates Fail

| Failure | Recovery |
|---------|----------|
| Headlines all < B-tier | QA auto-fix loop (max 3). If still failing, revisit offer promise. |
| Congruency fails PC2 | Body copy doesn't deliver headline promise. Rewrite body, not headline. |
| Congruency < 75% | Review which tests fail. Usually a shared context issue — check audience-product.md population. |

## Error Recovery Quick Reference

| Problem | Stage | Action |
|---------|-------|--------|
| < 3 competitors found | 1 | Broaden search terms or try adjacent categories |
| Thin VOC (< 5 categories) | 1 | Rerun meta-prompt with adjusted emphasis |
| < 3 competitor assets | 2A | Collect more before proceeding (hard minimum) |
| < 8 qualified habitats | 2B | Check scraper configs, expand date ranges |
| < 200 VOC items | 2B | Acceptable if high quality. Flag in corpus health audit. |
| < 5 viable angles | 2B | Thin VOC or saturated market. Consider adjacent segments. |
| No angle passes hard gates | 2B | Relax evidence threshold or expand VOC collection |
| All variants < 5.5 score | 3 | Max 2 iterations, then HUMAN_REVIEW |
| Copy scoring fails | 4 | Check shared context population first, then rewrite |
