## 0) Summary

Build a unified theme and component layer that matches the Base UI documentation site vibe (typography-forward, soft neutrals, subtle borders/shadows) while using Base UI React primitives for behavior/accessibility. Replace the tab-based demo feel with a platform-grade shell (sidebar + header) and ensure overlays (Dialog/Popover/Menu) animate and stack correctly across browsers (including iOS Safari).

## 1) Goals

1) Single global theme with tokens for typography, color, radius, shadow, spacing, mirroring Base UI docs aesthetic.  
2) Fonts aligned with the docs: one sans + one mono, deterministic imports, tokenized via CSS variables.  
3) `src/ui/*` wrappers over Base UI primitives so pages stop hand-styling (Button, Field, Table, Dialog, Tabs, Badge, Toast, Menu/Popover).  
4) Layout upgrade to sidebar + header shell; consistent page headers, cards, tables, forms.  
5) Overlay correctness: portal stacking via `isolation: isolate`, body positioning for mobile Safari, data-attribute-driven open/close animations.

## 2) Non-goals

* Pixel-perfect clone of base-ui.com.  
* Building a full design system beyond current app needs.

## 3) Requirements

* **Theme foundation**: `src/styles/theme.css` for tokens (fonts, sizes, colors, radii, shadows, spacing); `src/styles/globals.css` for base element defaults; `src/styles/fonts.css` for font imports. Tailwind config should map utilities to CSS variables (`--font-sans`, `--font-mono`, surfaces, borders, text-muted, accent).  
* **Typography parity**: Audit Base UI docs to capture sans/mono stacks, size scale, line heights, weights, feature settings. Codify in `docs/ThemeAudit.md`.  
* **UI kit**: Wrapper components with default styling/spacing and states; Dialog wrapper uses Base UI data attributes for animations and consistent backdrop/popup layout with action rows.  
* **Layout**: Sidebar navigation (Clients, Campaigns, Swipes/Library, Workflows, Tasks), top header with product identity/search/user menu, page headers with primary actions.  
* **Overlays**: Root container uses `isolation: isolate`; `body { position: relative; }` for iOS Safari; dialogs/menus/popovers share z-index tokens and transitions.  
* **Page refactors**:
  - Clients: structured form sections, labels/helper text, consistent spacing, toast/alert pattern for messages.  
  - Campaigns: standardized table/list rows, empty states matching theme.  
  - Swipes/Library: card grid with normalized typography and shared surface tokens.  
  - Workflows: row actions via Menu, Stop confirmation Dialog, better logs with mono font, approvals flow ready.

## 4) Acceptance criteria

* Single sans + mono font applied app-wide via variables; headings/labels/body text consistent.  
* All surfaces/borders/shadows/radii derive from tokens; tweaking `theme.css` shifts the whole UI.  
* Focus rings visible/consistent; Dialogs/overlays stack correctly and animate via Base UI state attributes; backdrops cover viewport on mobile Safari.  
* Pages primarily compose `src/ui/*` wrappers instead of ad-hoc styling.

## 5) Phases

* **Phase 0**: Theme audit of base-ui.com, record in `docs/ThemeAudit.md`.  
* **Phase 1**: Fonts + theme/globals CSS; Tailwind wired to tokens; root isolation/backdrop fixes.  
* **Phase 2**: UI kit wrappers refreshed with theme defaults and overlay animations.  
* **Phase 3**: App shell/layout updates (sidebar/header, page headers).  
* **Phase 4**: Page refactors (Clients, Campaigns, Swipes/Library, Workflows) using the kit.
